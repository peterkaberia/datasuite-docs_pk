# Modo portátil

O DataSuite oferece suporte ao [modo portátil](https://en.wikipedia.org/wiki/Portable_application).
Esse modo permite que todos os dados criados e mantidos pelo DataSuite fiquem armazenados próximos ao próprio aplicativo,
permitindo que sejam movidos entre diferentes ambientes.

O modo portátil é suportado no download ZIP para Windows.
Consulte a [página de download](/pt/downloads) para encontrar o arquivo `.zip` correto para sua plataforma.

> [!IMPORTANT]
>
> Não tente configurar o modo portátil em uma instalação proveniente dos instaladores Windows User
> ou System. O modo portátil é suportado apenas no arquivo ZIP (.zip) do Windows.
> Observe também que o arquivo ZIP do Windows não oferece suporte a atualização automática.

## Ativar o modo portátil

### Windows

Depois de descompactar o download do DataSuite, crie uma pasta `data` dentro da pasta do DataSuite:

```
|- DataSuite-win32-x64-1.0.0
|  |- Suite.exe
|  |- data
|  |- bin
|  |  |- code
|  |  |- ...
|  |- ...
```

A partir desse momento, a pasta `data` será usada para armazenar todos os dados do DataSuite,
incluindo estado de sessão, preferências, etc.

A pasta `data` pode ser movida para outras instalações do DataSuite.
Isso é útil ao atualizar sua versão portátil do DataSuite,
caso em que você pode mover a pasta `data` para uma versão mais recente extraída do DataSuite.

## Atualizar o DataSuite portátil

No Windows, você pode atualizar o DataSuite copiando a pasta `data`
para uma versão mais recente do DataSuite.

## Migrar para o modo portátil

Você também pode migrar uma instalação existente para o modo portátil.

### Windows

1. Baixe a distribuição ZIP do DataSuite para sua plataforma.
2. Crie a pasta `data` conforme descrito acima.
3. Copie o diretório de dados do usuário `DataSuite` para `data`
   e renomeie-o para `user-data`.

   Windows location:
   %APPDATA%\DataSuite

Exemplo:

```
|- DataSuite-win32-x64-1.84.2
|  |- DataSuite.exe
|  |- data
|  |  |- user-data
|  |  |  |- ...
|  |- ...
```

## Diretório TMP

Por padrão, o diretório TMP continua sendo o diretório do sistema mesmo no modo portátil,
pois nenhum estado é mantido nele.

Se você quiser que o diretório TMP também esteja dentro do seu diretório portátil,
você pode criar uma pasta `tmp` vazia dentro da pasta `data`.
Enquanto existir uma pasta `tmp`, ela será usada para armazenar dados TMP.
