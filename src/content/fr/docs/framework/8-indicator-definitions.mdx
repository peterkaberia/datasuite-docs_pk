## Définitions des indicateurs

La Data Suite normalise les définitions des indicateurs dans les systèmes d'information de gestion de la santé (SIGS / HMIS) pour permettre une analyse cohérente d'un pays à l'autre. Cette section précise exactement quelles variables du SIGS correspondent aux indicateurs de santé de la reproduction, de la mère, du nouveau-né, de l'enfant et de l'adolescent (SRMNIA / RMNCAH) — évitant ainsi toute mauvaise interprétation et garantissant que les données de vos établissements alimentent les bons numérateurs et dénominateurs. Des définitions normalisées permettent également des comparaisons entre les pays et dans le temps, rendant mesurables les progrès mondiaux vers la couverture sanitaire universelle.

Trois catégories de données organisent les exigences : les données administratives (inventaire des établissements, personnel), les données de population (estimations démographiques, taux de survie) et les données de service (événements signalés par les établissements, stratifiés par indicateur et niveau administratif). Étant donné que les structures des SIGS des pays varient considérablement, cette référence vous permet d'extraire ou de mapper systématiquement vos données vers les noms de variables standardisés attendus par la Data Suite. Le respect strict de ces définitions est fondamental pour une analyse fiable.

## Définitions des indicateurs

Pour la normalisation transnationale de l'analyse, cette section décrit les indicateurs/variables extraits du SIGS des pays, qui sont requis pour l'analyse Datasuite.  
Les données sont regroupées dans les catégories principales suivantes :

- **Données administratives (Admin Data)**
- **Données de population (Population Data)**
- **Exhaustivité des rapports (Reporting Completeness)**
- **Données de service 1, 2 et 3 (Service Data)**
- **Vérifications des données (Data Checks)**

---

### Données administratives (Admin Data)

Les variables suivantes décrivent l'infrastructure du système de santé et le personnel.

| Nom de la variable | Description |
|--------------|------------|
| `Num_order` | Nº |
| `District_name` | District |
| `First_admin_level` | Région |
| `Country` | Le nom de votre pays |
| `Number_hospitals` | Nombre total d'hôpitaux (selon la définition du pays) |
| `Number_hcenters` | Nombre total de centres de santé (non-hôpitaux ; selon la définition du pays) |
| `Total_number_health_facilities` | Nombre total d'établissements de santé (tous types/catégories confondus) |
| `Number_pfacilities_profit` | Nombre total d'établissements de santé privés à but lucratif |
| `Number_pfacilities_nonprofit` | Nombre total d'établissements de santé privés à but non lucratif |
| `Total_physicians` | Nombre total de médecins / médecins-chefs |
| `Total_nurses_midwives` | Nombre total d'infirmières / sages-femmes |
| `Total_clinical_officers` | Nombre total d'agents cliniques / assistants médicaux |
| `Total_nonclinique_physicians` | Nombre total de médecins non cliniciens |
| `Number_hospital_beds` | Nombre de lits d'hôpital |

### Vérifications des données (Data Checks)

Variables de diagnostic automatisées utilisées pour la validation de l'exhaustivité et de la cohérence entre les feuilles.

| Nom de la variable | Description |
|--------------|------------|
| `District` | Nom du district |
| `# of occurrences in 'Admin data'` | Décompte de vérification de la présence du district dans la feuille de données administratives (Admin Data) |
| `Is the district missing in 'Population_data'?` | Indicateur de statut vérifiant si le district est manquant dans les données de population (Population Data) |
| `# of occurrences in 'Population_data'?` | Décompte de vérification de la présence du district dans les données de population (Population Data) |
| `Is the district missing in 'Reporting_completeness'?` | Indicateur de statut vérifiant si le district est manquant dans l'exhaustivité des rapports (Reporting Completeness) |
| `# of occurrences in 'Reporting completeness'` | Décompte de vérification de la présence du district dans l'exhaustivité des rapports (Reporting Completeness) |
| `Is the district missing in 'Service_data_1'?` | Indicateur de statut vérifiant si le district est manquant dans les données de service 1 (Service Data 1) |
| `# of occurrences in 'Service_data_1'` | Décompte de vérification de la présence du district dans les données de service 1 (Service Data 1) |
| `Is the district missing in 'Service_data_2'?` | Indicateur de statut vérifiant si le district est manquant dans les données de service 2 (Service Data 2) |
| `# of occurrences in 'Service_data_2'` | Décompte de vérification de la présence du district dans les données de service 2 (Service Data 2) |
