# Mapping Configuration

## Overview

Mapping defines **what data to extract** from DHIS2 and **how it is structured** for analysis.
It is the core step in the Data Extractor workflow:
- Defines indicators
- Links DHIS2 data sources
- Standardizes outputs for Datasuite apps

## Mapping Workflow

1. Create a new mapping
2. Define metadata (name, description, mapping mode)
3. Configure indicators
4. Map DHIS2 data sources
5. Save and reuse mapping


## 1. Mapping Manager

![Mapping Manager](/images/extractor_mapping_manager.jpg)

The **Mapping Manager** is the central place to:
- View all saved mappings
- Search mappings by name or ID
- Create a new mapping

### Key Actions

- **New Mapping** → Start a new configuration
- **Search bar** → Locate existing mappings
- **Actions column** → Edit, download clone or delete the selected mapping

## 2. Creating a New Mapping

![New Mapping](/images/extractor_mapping_newmap.jpg)

<Steps>
### Click **New Mapping**

### Configure the mapping 
Define the mapping name, Description and the [mapping mode] (#mapping-mode)
<Callout type="important">
  **Important:** Use a clear naming convention
  Example:
    Mapping name: CAM2026_CountryName
    Description: MNH Indicators for CAM 2026 Workshop
</Callout>

### Select the [Mapping mode] (#mapping-mode)
There are currently two mapping modes:
- Countdown → This mode has a predefined list of MNCH indicators and structure as in the [harmonized data template] (/en/resources)
- Custom → Fully user-defined configuration, where the user can download any data that they have access rights to

<Callout type="warning">
  **Warning:** Changing the mapping mode will reset your progress if the mapping is not saved.
     Always save your mapping before switching modes
</Callout>
